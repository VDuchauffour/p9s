Commit: 7d460a1 feat(core): async event loop with EventStream and tokio::select!

Evidence — src/lib.rs main event loop using tokio::select!:

    loop {
        tokio::select! {
            biased;
            Some(event) = rx.recv() => {
                match event {
                    AppEvent::Tick => {
                        tui.terminal.draw(|frame| {
                            ui::render(frame, &app);
                        })?;
                    }
                    AppEvent::Key(key) => {
                        app.handle_key(key, &tx);
                        if app.quit {
                            break;
                        }
                    }
                    // ... other event variants
                }
            }
            _ = tokio::signal::ctrl_c() => {
                break;
            }
        }
    }

    tui.leave()?;
    Ok(())

Exit paths:
1. AppEvent::Key('q') sets app.quit = true -> break loop
2. tokio::signal::ctrl_c() -> break loop
3. Both paths call tui.leave() for terminal cleanup

Three background tasks spawned:
- spawn_polling_task: API resource polling every N seconds
- spawn_event_task: crossterm EventStream for keyboard input
- spawn_tick_task: 33ms render tick (30 FPS)

Clean shutdown verified: all tasks dropped when channel closes.
